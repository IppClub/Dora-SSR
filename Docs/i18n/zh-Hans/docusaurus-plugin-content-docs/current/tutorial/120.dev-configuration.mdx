## 2. 构建游戏引擎运行时

&emsp;&emsp;请选择你想要构建的目标平台。

<Tabs groupId="platform-select">
<TabItem value="windows" label="Windows">

1. 安装 **Rust**、**xmake**，以及带有 MSBuild 工具的 **Visual Studio Community 2022**。

2. 先构建 bgfx 依赖库，这一步与 GitHub Actions 保持一致。
	```bat
	Tools\build-scripts\build_lib_bgfx_windows.bat
	```

3. 构建 Windows 运行时与解决方案。
	```bat
	Tools\build-scripts\build_windows.bat
	```

4. 如果需要本地调试，可在 Visual Studio 中打开 **Projects/Windows/Dora.sln**，运行 `Debug` 配置。

</TabItem>
<TabItem value="macos" label="macOS">

1. 安装 **Rust**、**xmake** 和最新版 **Xcode**。

2. 先构建 bgfx 依赖库，与 CI 流程保持一致。
	```sh
	Tools/build-scripts/build_lib_bgfx.sh macos
	```

3. 生成 Lua 绑定。
	```sh
	cd Tools/tolua++
	./build.sh
	```

4. 为 macOS 构建 Rust 运行时并复制静态库。
	```sh
	cd Source/Rust
	rustup target add x86_64-apple-darwin
	cargo build --target x86_64-apple-darwin
	cp target/x86_64-apple-darwin/debug/libdora_runtime.a lib/macOS/libdora_runtime.a
	```

5. 本地构建并运行应用。
	```sh
	xcodebuild ARCHS=x86_64 ONLY_ACTIVE_ARCH=NO -project Projects/macOS/Dora.xcodeproj -target Dora -configuration Debug
	```

6. 如果需要 IDE 调试，可在 Xcode 中打开 **Projects/macOS/Dora.xcodeproj**，运行 `Dora` target。

</TabItem>
<TabItem value="ios" label="iOS">

1. 安装 **Rust**、**xmake** 和最新版 **Xcode**。

2. 先构建 bgfx 依赖库。
	```sh
	Tools/build-scripts/build_lib_bgfx.sh ios
	```

3. 生成 Lua 绑定。
	```sh
	cd Tools/tolua++
	./build.sh
	```

4. 为 iOS 模拟器构建 Rust 运行时。
	```sh
	cd Source/Rust
	rustup target add aarch64-apple-ios-sim
	cargo build --target aarch64-apple-ios-sim
	cp target/aarch64-apple-ios-sim/debug/libdora_runtime.a lib/iOS-Simulator/libdora_runtime.a
	```

5. 构建模拟器目标。
	```sh
	xcodebuild ARCHS=arm64 ONLY_ACTIVE_ARCH=NO -project Projects/iOS/Dora.xcodeproj -configuration Debug -target Simulator -sdk iphonesimulator
	```

6. 如果需要 IDE 调试，可在 Xcode 中打开 **Projects/iOS/Dora.xcodeproj**，运行 `Simulator` target。

</TabItem>
<TabItem value="android" label="Android">

:::tip
Android 本地构建除了 **Android Studio** 外，还需要 **JDK 17**。
:::

1. 安装 **Rust**、**xmake** 和最新版 **Android Studio**。

2. 先手动生成 Lua 绑定。
	```sh
	# 在 Ubuntu 系统上构建
	sudo apt-get install lua5.1
	sudo apt-get install -y luarocks
	sudo luarocks install luafilesystem
	cd Tools/tolua++
	lua tolua++.lua

	# 在 macOS 系统上构建
	cd Tools/tolua++
	./build.sh

	# 在 Windows 系统上构建
	cd Tools\tolua++
	build.bat
	```

3. 构建 Android 平台的 bgfx 依赖库。
	```sh
	Tools/build-scripts/build_lib_bgfx.sh android
	```

4. 构建 Android 各 ABI 对应的 Rust 运行时。
	```sh
	Tools/build-scripts/build_lib_android.sh
	```

5. 本地构建 Android 应用。
	```sh
	cd Projects/Android/Dora
	./gradlew assembleDebug
	```

6. 如果需要 IDE 调试，可在 Android Studio 中打开 **Projects/Android/Dora**，运行 `app` 模块。

</TabItem>
<TabItem value="linux" label="Linux">

:::tip
Linux 本地构建还会用到 `pkg-config` 以及 `dbus-1` 的开发包；通常桌面版 Linux 环境已经安装好它们，一般不需要额外处理。
:::

1. 安装 **Rust** 和 **xmake**。

<Tabs groupId="linux-distribution-select">
<TabItem value="ubuntu" label="Ubuntu/Debian">

3. 安装依赖包。
	```sh
	sudo apt-get update
	sudo apt-get install -y libsdl2-dev libgl1-mesa-dev libx11-dev lua5.1 luarocks
	```

4. 手动生成 Lua 绑定。
	```sh
	sudo luarocks install luafilesystem
	cd Tools/tolua++
	lua tolua++.lua
	```

</TabItem>
<TabItem value="arch-linux" label="Arch Linux">

3. 安装依赖包。
	```sh
	sudo pacman -S lua51 luarocks sdl2 openssl gcc make cmake --needed
	# 因为lua的版本必须是5.1,你需要使用lua5.1而不是最新的lua
	# 最简单的方法是用ln创建一个软链接
	sudo ln -s /usr/bin/lua5.1 /usr/local/bin/lua
	```

4. 手动生成 Lua 绑定。
	```sh
	sudo luarocks --lua-version 5.1 install luafilesystem
	cd Tools/tolua++
	lua5.1 tolua++.lua
	```

</TabItem>
</Tabs>

2. 构建 bgfx 依赖库，这一步与 Linux GitHub Action 保持一致。
	```sh
	cd Source/3rdParty/bgfx
	xmake f -p linux -a x86_64 -m release -y
	xmake -v
	```

3. 运行当前架构的编译脚本。

	```sh
	cd Projects/Linux
	make
	```

4. 运行生成的软件。
	```sh
	cd Assets
	../Projects/Linux/build/dora-ssr

	# 或者用命令行参数指定资源目录
	./Projects/Linux/build/dora-ssr --asset Assets
	```

</TabItem>
</Tabs>

## 3. 构建 Web IDE

1. 编译并运行 Dora SSR 引擎。
2. 安装最新版的 **Node.js** 和 **pnpm**。
3. 初始化 Dora Dora 编辑器。使用 `pnpm start` 进入开发模式，或用 `pnpm build` 生成 Web IDE 发布文件并同步到 `Assets/www`。

	请选择你的平台。

<Tabs groupId="platform-select">
<TabItem value="macos" label="macOS">

	```sh
	cd Tools/dora-dora && pnpm install
	pnpm start
	```

	```sh
	cd Tools/dora-dora
	pnpm build
	```

</TabItem>
<TabItem value="linux" label="Linux">

	```sh
	cd Tools/dora-dora && pnpm install
	pnpm start
	```

	```sh
	cd Tools/dora-dora
	pnpm build
	```

</TabItem>

	```sh
	cd Tools\dora-dora && pnpm install
	pnpm start
	```

	```sh
	cd Tools\dora-dora
	pnpm build
	```
</TabItem>
