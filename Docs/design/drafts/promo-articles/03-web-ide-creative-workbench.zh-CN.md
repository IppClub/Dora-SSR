# 浏览器里的创作工坊：Dora Web IDE 为什么不只是编辑器

> 「Dora SSR 2026 开发纪事」系列 · 第 3 篇 / 共 10 篇

## 一个小工具间，而不是一座办公楼

&emsp;&emsp;如果只看外表，Dora 的 Web IDE 很容易被误认为“浏览器里的代码编辑器”。它有文件树，有 Monaco 编辑器，有运行按钮，有日志输出，看起来像许多在线 IDE 的轻量版本。但 Dora Web IDE 真正想做的事情并不是把桌面 IDE 复制到浏览器里，而是在游戏引擎旁边搭一个小工具间。

&emsp;&emsp;工具间和办公楼不同。办公楼强调齐全、规范、层级分明；工具间强调伸手能拿到东西。你正在调动作，就应该能打开动作编辑器；你正在处理碰撞体，就应该能看到形状是否合法；你正在写剧情脚本，就不应该先跳出环境去找另一个工具。Dora Web IDE 的价值，恰恰在这些靠近创作现场的小入口里。

&emsp;&emsp;游戏开发不是只写代码。它总是在代码、资源、数据和运行效果之间来回摆动。一个引擎如果只给你脚本编辑器，很多工作就会掉到外部工具里；外部工具越多，创作状态被打断的次数就越多。Dora Web IDE 想做的是把这些小工作台尽量收进同一个浏览器窗口。

## 资源编辑器是创作的手感

&emsp;&emsp;今年 Web IDE 里许多工作都围绕资源编辑器和诊断体验展开。Action 编辑器、Body 编辑器、Particle 编辑器、Yarn 编辑器各自处理不同类型的创作材料。它们不是为了证明 Dora 有多少面板，而是为了让开发者少在“格式”和“意图”之间来回翻译。

&emsp;&emsp;比如粒子配置出错时，最糟糕的体验不是失败本身，而是不知道哪一项失败。一个 XML 里少了数字、标签不认识、属性格式不对，如果只给一句“解析失败”，开发者就必须像在黑暗里摸螺丝。Body 编辑器也类似。多边形顶点不够、面积为零、自相交、分解失败，这些都是资源层面的具体问题。工具把它们说清楚，开发者才知道下一步要调整哪里。

&emsp;&emsp;这类诊断有点像工坊墙上的刻度尺。它不会替你做设计，但它会告诉你这块木料是不是歪了、这个角度是不是超过了限制。创作工具最好的状态不是不停弹出教程，而是在你手碰到材料时，给出足够及时、足够具体的反馈。

&emsp;&emsp;更重要的是，这些编辑器和正在运行的游戏就在同一个浏览器里。你保存一个改好的碰撞体，重新运行，看到的就是同一台设备上立刻生效的结果——没有桌面工具链要同步，没有部署步骤要等。工坊之所以是工坊，不在于工具多，而在于手边的材料和成品之间没有漫长的搬运距离。

## 浏览器不是妥协，而是连接方式

&emsp;&emsp;很多人会把 Web IDE 理解成“没有桌面 IDE 强，所以退而求其次放到浏览器里”。Dora 的情况正好相反。浏览器不是妥协，而是连接不同设备的方式。引擎可以在 Android、Linux、macOS 或掌机上运行，开发者从局域网里的任意浏览器接入。浏览器在这里像一根可插拔的工作灯，照向正在运行 Dora 的设备。

&emsp;&emsp;这种设计天然适合 Dora 的真机开发方向。你不必在每台目标设备上安装完整桌面工具，也不必每次都通过复杂部署流程同步项目。只要设备上的 Dora 服务跑起来，Web IDE 就能成为入口。它把编辑器、资源工具、构建、日志和 Agent 接到同一个现场。

&emsp;&emsp;当然，浏览器也会带来自己的麻烦。移动端 WebView、剪贴板权限、输入法行为、焦点管理、文件拖拽、窗口尺寸、滚动区域，这些都是桌面应用里不会以同样方式出现的问题。但 Dora 接受这些麻烦，因为浏览器换来的不是“轻量”两个字，而是一种更开放的连接方式。

## 工具越小，越要懂场景

&emsp;&emsp;Dora Web IDE 不是要和大型商业编辑器比规模。它更像一组贴近 Dora 引擎的专用工具。正因为它小，才更需要懂场景：它知道 Dora 的脚本语言，知道 DoraX 的 TSX 结构，知道 Yarn 文件不是普通文本，知道粒子和碰撞体有自己的约束，知道构建不是单纯调用 tsc，知道错误信息应该尽量回到创作者能理解的位置。

&emsp;&emsp;这也是开源游戏引擎很值得做 Web IDE 的原因。大型通用 IDE 很难为每个小引擎深度定制所有体验，而 Dora 可以直接把自己的知识写进工具里。它不需要面面俱到，只需要在 Dora 用户最常经过的路口放好路标。

&emsp;&emsp;当 Web IDE 真的做到这一点时，它就不再是“浏览器里的编辑器”，而是 Dora 的创作工坊。开发者在里面写代码、改资源、看诊断、跑游戏、问 Agent，很多事情都围绕同一个项目现场发生。这种集中并不是为了封闭，而是为了减少创作时的切换成本。这是工坊和办公楼最根本的区别：办公楼追求齐全，工坊追求顺手——而 Dora Web IDE 选择做后者。
