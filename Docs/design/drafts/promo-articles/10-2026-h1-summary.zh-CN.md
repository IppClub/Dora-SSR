# Dora 2026 上半年：一个开源游戏引擎如何在 AI 参与下长出新的工作方式

> 「Dora SSR 2026 开发纪事」系列 · 第 10 篇 / 终篇

## 今年的主线其实只有一条

&emsp;&emsp;如果把 Dora SSR 今年以来的开发进展摊开看，会看到很多方向：Android 真机开发、Web IDE、Coding Agent、Dora CLI、DoraX、TypeScript 声明、多语言生态、Linux ARM64、A10mini、PPA、发布同步、资源编辑器、输入焦点、文档和教程。它们看起来像许多分散任务，但背后有一条越来越清晰的主线：Dora 正在从“一个能运行游戏的开源引擎”，变成“一个能陪创作者完成工程过程的开放开发环境”。

&emsp;&emsp;这条主线里，AI 是今年最大的变量。不是因为 Dora 在界面上多了一个 AI 按钮，而是因为 Dora 自身的开发方式已经被 Codex 改变。很多引擎相关工作由 Codex 参与完成：读源码、定位问题、修改实现、跑构建、查设备、同步文档、验证发布、整理会话经验。AI 不是宣传语，而是实际进入了维护过程。

&emsp;&emsp;因此，如果要写 Dora 今年的故事，我不想只写“新增了哪些功能”。更有意思的是：一个开源游戏引擎在 AI 参与以后，哪些原本很难持续推进的细节开始被推进？哪些工具设计因为 AI 的参与而改变？哪些面向人的体验，也同时变成了面向 AI 的工程接口？

## 从功能列表到工作流

&emsp;&emsp;过去介绍引擎，常常会列功能：渲染、动画、物理、脚本、编辑器、平台支持。功能列表当然必要，但它不完全解释开发者真实体验。一个人做游戏时，更多时间花在工作流里：启动项目、写脚本、改资源、看错误、切设备、调输入、发布版本、查文档、修小问题。

&emsp;&emsp;Dora 今年很多进展，正是围绕工作流展开。CLI 的 doctor/status 让项目状态可询问；Web IDE 的 active-tab TypeScript 诊断让编辑反馈更安静；DoraX 让场景结构更可读；Agent 工具注册表让 AI 调用工具更有边界；A10mini 适配让引擎进入真实掌机；PPA 和 Ports 让 Linux 设备用户更容易启动；发布同步流程让多平台包更可信。

&emsp;&emsp;这些事情单独看不一定惊人，但它们合起来改变了 Dora 的形状。Dora 不再只是“你可以用它写游戏”，而是越来越像“你可以在它提供的环境里把游戏开发过程走下去”。

## AI 让小项目有机会处理更多细节

&emsp;&emsp;开源小项目最难的地方不是没有想法，而是维护注意力稀缺。一个人或小团队要同时处理核心功能、平台适配、文档、发布、用户反馈和工具体验，很容易把许多“小但烦”的问题放到以后。可这些小问题对用户来说并不小：WebView 不能粘贴、status 太啰嗦、doctor 不够详细、手柄焦点穿透、TypeScript 背景标签红线乱跳、发布包不是最新。

&emsp;&emsp;Codex 参与以后，这些细节有了被持续清理的可能。AI 不会自动让项目变大团队，但它可以补上一部分工程耐心。它愿意读长文件，愿意跑重复检查，愿意在失败后继续缩小范围，愿意把一次会话的经验写成下一次可复用的规则。这对 Dora 这样的项目非常关键。

&emsp;&emsp;今年 Dora 的许多故事都不是“AI 一次解决复杂问题”，而是“AI 陪着把问题拆到可以解决”。A/B 按键穿透就是这样：先挡事件不够，再追状态轮询；TypeScript 诊断也是这样：先统一路径，再收窄模型，再分离实时检查和构建检查；发布同步也是这样：先确认 live 源头，再下载、验证、上传、读回。AI 的价值体现在过程里。

## TypeScript 成为 AI 协作的中轴

&emsp;&emsp;Dora 坚持多语言生态，但今年 TypeScript 的地位变得更突出。原因很实际：AI 需要反馈，而 TypeScript 的反馈密度很高。类型声明、编辑器补全、语义诊断、TSX 结构、转译检查，这些都能帮助 Codex 生成更可验证的代码。相比完全靠运行时发现问题，TypeScript 能提前给出许多边界。

&emsp;&emsp;这并不意味着 Dora 要放弃 Lua、YueScript、Teal、Wa、Rust 或 C#。多语言仍然是 Dora 的开放姿态。但在 AI 协作的主路径上，TypeScript 目前确实更像一条中轴路。它让人和 AI 可以围绕同一套类型语言讨论代码，也让 Web IDE 和 Agent 有更多结构化信息可以利用。

&emsp;&emsp;Node 类型声明的讨论就是一个象征。用户想写更自然的 TypeScript，旧 API 又要兼容，生成规则还要尊重。这个问题不是宏大功能，却说明 Dora 未来要面对的方向：多语言门槛可能会被 AI 抹平一部分，但底层类型和声明质量会变得更重要。

## 真机、Web IDE 和 Agent 汇合在一起

&emsp;&emsp;Dora 今年另一个明显趋势，是真机开发、Web IDE 和 Agent 正在汇合。真机让引擎进入真实设备，Web IDE 让浏览器成为开发入口，Agent 让 AI 进入项目上下文。三者组合起来，就出现了一种很有 Dora 气质的画面：一台掌机或手机运行 Dora，浏览器打开 Web IDE，开发者和 Codex 一起在真实设备旁边修改、验证、排查。

&emsp;&emsp;这不是传统桌面大编辑器的路线，也不是纯云 IDE 的路线。它更轻、更贴近设备、更适合开源创作者折腾。Dora 可以在电脑上用，也可以塞进小设备里；可以人手写代码，也可以让 Agent 帮忙处理工程任务；可以通过 CLI 检查状态，也可以通过 Web IDE 直接进入项目。

&emsp;&emsp;这条路线还很早，但它已经形成了自己的特点：不把真实设备放到最后，不把 AI 放在旁边，不把浏览器只当编辑器外壳。Dora 想把这些东西放在同一个创作现场里。

## 一个开源游戏引擎的新叙事

&emsp;&emsp;如果要对外推广 Dora，我觉得今年不应该只强调“功能更多了”。更好的叙事是：Dora 正在探索 AI 时代开源游戏引擎的新工作方式。它不是只给创作者 API，而是给创作者一个可以在真实设备上运行、在浏览器里开发、由 AI 参与维护和协作的环境。

&emsp;&emsp;这个叙事可以拆成九个小主题，也是本系列前面九篇文章：

1. **把游戏引擎装进设备里**——真机不是发布的终点，而是开发的工作台。
2. **游戏引擎里的 Coding Agent**——当通用 agent 已经不稀奇，难的是让它懂这个引擎。
3. **浏览器里的创作工坊**——Web IDE 不复制桌面 IDE，而在引擎旁边搭工具间。
4. **Web IDE 的细节战**——工具好不好用，藏在红线分层和输入焦点里。
5. **DoraX 和 TSX**——把场景树写成一张可读的描图纸。
6. **多语言、AI 和 TypeScript**——给创作者留很多扇门，TypeScript 是 AI 当前最稳的主路。
7. **Dora CLI**——一个开源引擎的入口应该像路标。
8. **从 GitHub 到掌机**——跨平台发布像一次港口调度。
9. **Dora 跑上 A10mini**——开源引擎的“最后一米”。

&emsp;&emsp;这些主题合起来，比单纯功能列表更能说明 Dora 的方向。它不是要和所有商业引擎正面比规模，而是要成为一个开放、轻便、可理解、能被 AI 放大的游戏创作环境。

## 最大的主题：用 AI 开发 Dora，再把经验做回去

&emsp;&emsp;今年 Dora 最值得记录的，不只是 Codex 帮忙写了多少代码，而是 Codex 改变了 Dora 处理问题的方式。它让许多原本容易被搁置的工程细节得到推进，也让 Dora 更清楚地意识到：未来的开发工具既要服务人，也要服务 AI。清晰的 CLI、准确的类型声明、可读的 TSX、明确的诊断、稳定的验证命令，都会成为 AI 协作的基础。

&emsp;&emsp;因此，Dora 的 AI 主题不应该写成简单的“我们接入了 AI”。更准确的说法是：Dora 正在用 AI 开发 Dora，再把这个过程中的经验做回 Dora。Codex 参与了设备适配、Web IDE、Agent、CLI、发布和语言绑定；这些经历又反过来塑造了 Dora 对 Agent、TypeScript、工具反馈和工作流的设计。这个循环，才是今年最大的主题。

## 想动手试试？

&emsp;&emsp;Dora SSR 可在 Android、Windows、Linux、macOS 上原生运行，打开引擎后用同一局域网里的任意浏览器连上 Web IDE 就能开始写第一个游戏——无需桌面工具链，也无需打包部署。安装与快速上手见 [官方文档](https://dora-ssr.net/docs/tutorial/quick-start)，最新版本在 [Releases](https://github.com/ippclub/Dora-SSR/releases/latest)，单独的 API 示例和完整 Demo 项目分别在 [Dora-Example](https://github.com/IppClub/Dora-Example) 与 [Dora-Demo](https://github.com/IppClub/Dora-Demo)。如果你也想在自己的设备上把想法跑起来，欢迎加入 [Discord](https://discord.gg/ZfNBSKXnf9) 或 QQ 群 512620381 一起聊。
