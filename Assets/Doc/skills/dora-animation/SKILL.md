# Dora Animation

Use Dora engine animation APIs. Do not use DOM/CSS/Canvas animation, browser `requestAnimationFrame`, or hand-written interpolation when Dora actions can express the behavior.

## Choose the Right API

| Need | Use |
| --- | --- |
| Move/scale/rotate/fade node properties | `ActionDef` creators + `node.perform()` / `node.runAction()` |
| Combine actions in order | `Sequence(...)` |
| Combine actions in parallel | `Spawn(...)` |
| Simple horizontal sprite sheet | `Sprite` + `Frame(...)` |
| Authored model/skeleton animation | `Model` / `Spine` / `DragonBone` / `Playable` + `.play()` |
| Generated image-sprite metadata, FPS, `.sprite.json` | Also use the `image-sprite` skill |

## Node Property Animation

```ts
import {
	Director, Sprite, Vec2,
	Move, Scale, Angle, Opacity,
	Sequence, Spawn, Ease,
} from "Dora";

const sprite = Sprite("Image/logo.png");
if (sprite) {
	sprite.addTo(Director.entry);
	sprite.perform(Sequence(
		Move(0.4, Vec2(0, 0), Vec2(200, 80), Ease.OutBack),
		Spawn(
			Scale(0.2, 1, 1.3, Ease.OutQuad),
			Opacity(0.2, 1, 0.6),
		),
		Spawn(
			Scale(0.2, 1.3, 1),
			Opacity(0.2, 0.6, 1),
		),
		Angle(0.3, 0, 360),
	));
}
```

Common signatures:

```ts
Move(duration: number, from: Vec2, to: Vec2, easing?: EaseFunc): ActionDef
Scale(duration: number, from: number, to: number, easing?: EaseFunc): ActionDef
ScaleX(duration: number, from: number, to: number, easing?: EaseFunc): ActionDef
ScaleY(duration: number, from: number, to: number, easing?: EaseFunc): ActionDef
Angle(duration: number, from: number, to: number, easing?: EaseFunc): ActionDef
Roll(duration: number, from: number, to: number, easing?: EaseFunc): ActionDef
Opacity(duration: number, from: number, to: number, easing?: EaseFunc): ActionDef
X/Y/Z(duration: number, from: number, to: number, easing?: EaseFunc): ActionDef
Delay(duration: number): ActionDef
Show(): ActionDef
Hide(): ActionDef
Sequence(...actions: ActionDef[]): ActionDef
Spawn(...actions: ActionDef[]): ActionDef
```

Use `perform(actionDef, loop?)` when the new action should clear currently running actions and start immediately. Use `runAction(actionDef, loop?)` when adding an action to the node action system. Both return the new action duration in seconds.

Common easing values: `Ease.Linear`, `Ease.OutQuad`, `Ease.InOutQuad`, `Ease.OutBack`, `Ease.InBack`, `Ease.OutBounce`, `Ease.OutElastic`.

## Sprite Sheet Frame Animation

For simple horizontal sprite sheets, use `Sprite` + `Frame()`. Do not hand-slice textures unless `Frame()` cannot represent the asset.

```ts
import { Director, Frame, Sprite } from "Dora";

const player = Sprite();
player.addTo(Director.entry);
player.runAction(Frame("hero.idle_front.png", 0.5), true);
```

Signature:

```ts
Frame(clipStr: string, duration: number, frames?: number[]): ActionDef
```

`Frame()` only works on `Sprite` nodes. If the asset is generated by the image-sprite editor and has `.sprite.json`, load the `image-sprite` skill for metadata/FPS/duration rules.

## Model / Spine / DragonBone / Playable

```ts
import { Director, Model, Sprite } from "Dora";

const character = Model("assets/character");
character.addTo(Director.entry);

character.speed = 1.0;
character.recovery = 0.2;
character.fliped = false;
character.play("idle", true);

const sword = Sprite("assets/sword.png");
character.setSlot("right_hand", sword);

character.onAnimationEnd((name, target) => {
	if (name === "attack") {
		target.play("idle", true);
	}
});

character.play("attack");
```

Known baseline methods/properties:

```ts
play(name: string, loop?: boolean): number
stop(): void
onAnimationEnd(callback: (name: string, playable: Playable.Type) => void): void
setSlot(name: string, item: Node.Type | undefined): void
getSlot(name: string): Node.Type | undefined
speed: number
recovery: number
fliped: boolean
current: string
lastCompleted: string
```

Use project-relative asset paths. Search Dora API before advanced animation APIs not listed here, such as custom `Action` objects, effects, editor/model serialization, complex slots, or keypoint workflows.
